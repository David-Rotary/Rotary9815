How the Chair Hub pages are built

A structure guide to the Membership Chair Hub and Public Image Chair Hub: what each part of the page is called, and which Google Sheet tab controls it. Written so you can point someone at the right element without guessing.

Architecture

One HTML file, one Google Sheet

Both hubs work the same way: a single self-contained HTML file holds the design and behaviour, and a Google Sheet holds every word, link, and list on the page. Nothing is hard-coded except the overall shape (which tabs exist, what a card looks like) — the words inside that shape all come from the sheet.

Google SheetOne tab per content type — Cards, ChecklistItems, ResourceLinks, and so on
→
Page loadsThe HTML fetches every tab over Google's public gviz/tq endpoint
→
Rendered liveJavaScript builds the hero, tabs, cards and planner from what it fetched

Because the fetch happens every time the page loads, editing a cell in the Sheet changes the live page the next time anyone opens it — no re-uploading the HTML, no rebuild step. The only thing wired into the HTML file itself is the Sheet's ID (CONFIG.SHEET_ID near the top of the <script> block) and the Sheet's sharing setting must be "Anyone with the link — Viewer".

Layout

Page anatomy, top to bottom

Every region of the page, in the order it appears, and whether it's fixed in the HTML or pulled from the Sheet.

Guided Flow panel
The dark blue strip with step text and Back / Next / Exit. Hidden until "Start Guided Flow" is clicked. Sheet-driven — text comes from GuideSteps.
Breadcrumb
The small "Rotary District 9815 · Hub Name · …" line above the hero. Fixed in HTML — not editable from the Sheet.
Hero
Navy gradient banner: eyebrow line, heading, subtext, and the two CTA buttons (Start Guided Flow / View Year Planner). Sheet-driven — text from Settings; Expand All / Collapse All buttons sit here too but are fixed in HTML.
Rainbow bar
The thin six-colour stripe under the hero. Fixed in HTML.
Menu pills (tab nav)
Overview / Attract / Onboard etc. — the row of clickable tabs below the hero. Fixed in HTML: the list of tabs and their icons is set in the page code, not the Sheet.
Stat row
The four or five number tiles at the top of the Overview tab (e.g. "7 Topic Areas"). Sheet-driven — from Stats.
Section heading
Each grey icon + title + subtitle block that introduces a group of cards (e.g. "Your Role as Membership Chair"). Sheet-driven — from Sections, one row per heading, grouped by which tab it belongs to.
Card grid → Card
The expandable white tiles under each section heading. Each card's icon, title, description and badge come from Cards; what's inside it when opened comes from several other tabs — see Card block types below. Sheet-driven.
Health Check sliders
The self-rating sliders section on the Overview tab. Sheet-driven — from HealthSliders. This section is special-cased by its section_id (sec_health) rather than holding cards.
Year Planner quarters
The four Q1–Q4 tiles on the Year Planner tab. Sheet-driven — quarter headings from Quarters, the action lines inside each from QuarterItems. Also special-cased by section_id (sec_planner_main).
Rule of thumb: if you can imagine wanting to reword it without touching code, it's in the Sheet. If it's structural — which tabs exist, the order of card blocks by default, the colour palette — it's in the HTML.

Cards, opened up

Card block types

When a card is expanded, its contents are built from whichever of these block types have matching rows for that card_id. A card can use several — e.g. links followed by a checklist followed by a tip.

BlockSheet tabWhat it renders
rlinksResourceLinksA stack of outbound link rows
matrixMatrixCellsA grid of small titled boxes
checklistChecklistItemsTickable checklist lines
teamgridTeamRoles (Public Image Hub only)Role tiles with icon, title, description
pipelinePipelineStepsNumbered step-by-step list
promptsPromptsHighlighted 💡 tip callout(s)

By default, blocks render in this order: rlinks → matrix → checklist → teamgrid → pipeline → prompts, skipping any type with no rows for that card. On the Public Image Hub, a card can override this by putting a comma list in its block_order cell on the Cards tab — e.g. checklist,rlinks — used for the one card ("Online Discovery & Google") where the checklist needed to come before its links.

Walkthrough

How Guided Flow is wired

The GuideSteps tab is a simple script: each row is one step, naming which page tab it lives on, which card it should pop open, and the text shown in the panel.

order
The sequence of steps — 1 through however many rows exist.
tab
Which top-level tab to switch to for this step — must match a tab key used in the page (e.g. overview, brand, planner).
card_id
Which card to open and highlight in gold — must exactly match a card_id on the Cards tab.
label / text
The small step counter label and the main panel copy shown while on that step.
Adding a step: add a row to GuideSteps with the next order number, the tab it should jump to, and the exact card_id of the card it should open. No code changes needed.
Day to day

Editing workflow

To…Do this
Reword somethingFind the text in the relevant tab and edit the cell directly
Reorder cards, links, or checklist itemsRenumber the order column for that group
Add a new cardAdd a row to Cards with a new, unique card_id, then add matching rows to whichever content tabs it needs, using that same card_id
Remove a card or itemDelete its row — just don't delete a card_id that other tabs still reference
Add a Guided Flow stepAdd a row to GuideSteps pointing at an existing tab and card_id
Change a card's badge or block colourUse one of: blue, gold, green, purple, red, teal — leave blank for the default

Changes take effect the next time the page is loaded — there's nothing to publish or rebuild. If a page ever fails to load content, it shows a plain-English error naming the likely cause (usually the Sheet ID or its sharing setting).
